# DSAce.dll

Клиентский хук для Freelancer, используемый в модах, работающих через обнаружение и взаимодействующих с серверным плагином `dsacesrv.dll` для FLHook.

- [Пример конфигурации `dsacesrv.cfg`](../../../../Binaries/bin-conf/flhook_plugins/dsacesrv.cfg)

---

## Описание

`DSAce.dll` — это **клиентский перехватчик (hook)**, который расширяет стандартный функционал клиента Freelancer и позволяет серверу управлять дополнительными игровыми механиками.

Обычно он используется в связке с:

- `dsacesrv.dll`
- FLHook-плагинами, которым нужно взаимодействие с клиентом

---

## Возможности

Этот хук добавляет в клиент следующие возможности:

- ограниченный **античит на стороне клиента**
- обновление **информационных карт**
- обновление **экономики под контролем сервера**
- **масштабирование турелей**
- поддержку **маскировки**
- поддержку **прыжкового привода**

---

## Когда он нужен

Этот хук **обязателен**, если вы используете один из следующих серверных плагинов:

- `base.dll` — плагин баз, принадлежащих игрокам
- `dock.dll` — плагин мобильной стыковки
- `cloak.dll` — плагин маскировки
- функции прыжка в `player control plugin`

Эти плагины отправляют команды и данные на клиент:

- либо через скрытые сообщения чата
- либо через серверный плагин `dsacesrv.dll`

Без `DSAce.dll` клиент просто не сможет корректно обрабатывать эти механики.

---

## Установка

Чтобы подключить хук, добавьте:

```text id="bqfzw8"
DSAce.dll
```

в секцию:

```ini id="q77n0p"
[Libraries]
```

файла:

```text id="phc1ie"
dacom.ini
```

## Исходный код

Полный исходный код доступен **не всем**.

Если нужна полная версия — необходимо запрашивать отдельно.

В комплекте доступны только неконфиденциальные части:

- `public_funcs.cpp`
- `Equipment_funcs.cpp`

---

## Античит

В плагине присутствуют функции античита на стороне клиента, но сам автор прямо отмечает:

> они не очень хороши, но это лучше, чем ничего

То есть это **не полноценная защита**, а скорее дополнительный базовый уровень контроля.

---

## Назначение

Этот модуль нужен в первую очередь для модов и FLHook-сборок, где реализованы:

- нестандартные игровые механики
- взаимодействие клиента и сервера через скрытые команды
- расширенные корабельные возможности
- серверные игровые системы, требующие клиентской поддержки
